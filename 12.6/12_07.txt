새로운 controller만들 때 스프링이 관리해야함[인지]
=> bean 스피링이 관리하는 객체
빈으로 등록 필요
빈으로 등록하는 annotation
- @Component [최상위] : bean으로 등록함

- @Controller[Component 상속]  controller 역할로 bean 등록
- @Repository[Component 상속] : DAO클래스 역할로 bean 등록
- @Service[Component 상속] : CMD 클래스 역할로 bean 등록
- @Configuration[Component 상속] : bean 등록 기능을 수행

Spring에서 bean으로 등록하는 3가지 방법
! bean으로 관리되는 객체는 singleton
1. annotation [내가 만든 클래스가 아닐 때 안됨]
	=> @Autowired	[D.I( dependency injection )]  
		=> otherAnnotationBean [ injection  할 Bean이 1개가 아닌 경우 => 500에러]
			=> 1.주입할 Bean 지정 변수명을 bean id로 지정한다.
				=> 변수명이 길거나 지정 : @Component("bean1") 어노테이션 옆에 bean id 지정
			=> 2. @AutoWired 어노테이션 밑에 @Qualifier(beanid) 어노테이션 사용

같은 타입 객체 여러개 불가능
new Some해서 쓸 수 밖에 없어서 지정해서 사용 불가능
